-일괄변경과 변경신청의 차이? (내가 보고 느낀 차이점)
1. 일괄변경 에서 변경신청을 하게되면, 변경하고자 하는 목록이 변경 후 값을 받아 신고하도록 되어있다. 하지만 변경신청은 변경하고자 하는 목록의
모든 데이터들의 변경 전 데이터와 바꾸고자하는 변경 후 데이터를 담도록 되어있다.
	=> 일괄변경은 변경 후 데이터를 다룸, 변경신청은 변경 전과 변경 후 데이터를 다룸

	=> 일괄변경은 기존 데이터를 변경하는 개념
	=> 변경신청은 기존사이트 데이터가 없을 수 있음(2017년 이전 건) -> 종이로 갖고있거나 데이터 베이스에 존재하지 않는 여러 매채로 신고내역을
	가질 수 있기 때문에 변경신청 건을 확인할 증명서를 필수 제출할 수 있도록 함.
		=> 위 내용은 회의 중 차장님께서 말씀주신 내용인데, 일괄변경의 기존 데이터 변경 개념은 변경신청에서 이전 데이터와 바뀔 데이터로 
		나뉘기 때문에 변경신청이 기존 데이터를 변경하는 개념이라고 생각된다.
		=> 일괄변경이 이전 데이터가 없기 때문에, 변경 후 건의 데이터만 작성하고, 필히 증명서를 첨부해야 하도록 되어있기 때문에 일괄변경과
		변경신청의 내용이 바뀐 것으로 예상됨.

	=> 일괄변경과 변경신청의 공통점은 신고내역이 관리자에게서 승인이 완료된 건에 대해서만 변경신청 할 수 있도록 함
