<암호화 관련 SELECT 로직>
중경 실행하고, 내가 맡고있는 '인사등록표'를 실행하였다. 그런데 무한 로딩 돌다가 콘솔창에는 SELECT문을 마지막으로 돌고, 'server no longer responding'
이라는 내용의 에러문이 발생했다. SERVER 에러인가 싶어 네트워크 관련된 블로그를 보게되었다. 네트워크의 IP주소를 바꾸라는 등, 여러 글이 있었지만 해답은 아니었다. 정답은 SELECT문에서 주민등록번호를 담당하는 곳에서 오류였다고 한다.

주민등록번호는 'crypto_dec ('normal',ST.IHIDNUM,'',0) /*주민등록번호*/'라고 되어있었다. 무엇이 문제인가?
-> 위의 문장은 암호화를 시도하는 동작이다. 하지만 이게 왜 문제인가? 
	-> 암호화가 되어있지 않은 상태에서 암호화를 시도하려는 동작이 돌게되어 오류가 발생한 것이다.
즉, 암호화가 되어있지 않은 상태의 쿼리를 돌아 암호화를 시도하면서 없는 값을 동작하려고 하다보니 오류가 발생한 것이다.

도현님께서 처리해주셨는데, UPDATE문으로 'ST.IHIDNUM'과 'crypto_dec ('normal',ST.IHIDNUM,'',0) /*주민등록번호*/'을 넣어주셔서 프로젝트에 문제 없이 동작할 수 있게 되었다.
